Выявленные проблемы
1. Отсутствие привязки пользователей к org_units (Главная причина)
Из 20 DCHS пользователей только 1 (dchs_almaty) привязан к org_unit
Все OCHS пользователи имеют org_unit_id = null
Это приводит к ошибке "org unit required" или "forbidden"
2. Недостаточно организационных единиц
В базе только 7 org_units вместо нужных 20+ регионов
Нужно создать все 20 регионов (DCHS) + районы (OCHS)
3. Формы показывают нули для DCHS
ДЧС Алматы (который привязан) запрашивает данные за "Республика Казахстан (Свод)" - это выходит за его область доступа
Фронтенд не ограничивает выбор региона по роли пользователя
4. Ошибка enum inspection_type: "preventive"
В базе: {scheduled, unscheduled, preventive_control, monitoring}
В коде используется preventive вместо preventive_control
5. Ролевая модель не применяется к фильтрам
Фронтенд позволяет выбрать любой регион независимо от роли
Нужно ограничить выбор согласно правилам:
admin - всё видит и редактирует
MCHS - только чтение всей РК
DCHS - видит и редактирует только свою область
OCHS - видит и редактирует только свой район
